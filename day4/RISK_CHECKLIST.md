# 企业知识库 AI 助手上线风险清单

## 1. 成本风险

### 风险

企业 AI 助手每次回答都可能调用大模型 API。  
如果员工频繁使用，token 成本可能持续增加。

成本来源包括：

- 用户问题输入 token
- 检索到的文档片段 token
- AI 输出 token
- embedding / 向量化成本
- 服务器和数据库成本

### 控制方式

- 控制 top_k，避免一次传入过多文档片段
- 控制 chunk 长度
- 对重复问题做缓存
- 对高频标准问题使用固定答案
- 限制用户每日调用次数
- 使用不同模型处理不同复杂度问题
- 记录每次调用的 token 消耗
- 定期分析调用量和费用

## 2. 延迟风险

### 风险

如果文档量大、检索慢或模型生成慢，用户等待时间会变长。

延迟来源包括：

- 文档检索耗时
- 大模型生成耗时
- 网络请求耗时
- 后端处理耗时
- 数据库查询耗时

### 控制方式

- 文档上传时提前读取、切分和入库
- 用户提问时只做检索，不重新处理全部文档
- 控制 top_k
- 使用缓存减少重复生成
- 给前端增加 loading 状态
- 后续支持流式输出
- 为接口设置超时和错误提示

## 3. 数据安全风险

### 风险

企业知识库可能包含员工信息、客户资料、财务数据、合同内容、部门流程等敏感信息。

如果权限控制不完善，AI 可能把不该公开的信息回答给无权限用户。

### 控制方式

- API Key 放在服务端 .env 中
- .env 不上传 GitHub
- documents/ 不上传 GitHub
- history.json 不上传 GitHub
- 增加用户登录
- 增加角色权限
- 增加部门文档隔离
- RAG 检索前先过滤用户可访问文档
- 对敏感信息做脱敏
- 保留操作日志和审计记录

## 4. 文档质量风险

### 风险

如果企业上传的文档过期、错误或重复，AI 会基于错误资料回答。

### 控制方式

- 文档上传只允许管理员操作
- 文档增加上传人、部门、版本、时间字段
- 定期清理过期文档
- 删除重复文档
- 对重要文档增加审核流程
- 用户反馈“没用”后进入复查

## 5. AI 幻觉风险

### 风险

AI 可能在资料不足时编造答案，误导员工。

### 控制方式

- prompt 中要求 AI 必须基于资料回答
- 没有明确资料时回答“无法确认”
- 展示参考来源
- 低相似度来源不强行展示
- 用户可以反馈“有用 / 没用”
- 高风险问题转人工处理
