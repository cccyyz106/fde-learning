## Day 3 结束复盘

1. 今天学会了什么？

今天我学习了 HTTP、API、GET、POST、JSON 和状态码。我理解了 API 是软件之间沟通的接口，HTTP 是前端和后端之间传递请求和响应的规则，JSON 是常见的数据格式，状态码用来表示请求结果。

2. 今天做出了什么？

今天我使用 Apifox 测试了公开 API。先用 GET 请求获取了一篇文章数据，状态码是 200；然后故意请求一个不存在的资源，观察到了 404；最后用 POST 提交了一段 JSON 数据，服务器返回 201 和新创建的数据 ID。

3. 今天最卡的地方是什么？

今天最卡的地方是刚开始不理解为什么要学 API，以及 GET、POST、JSON、状态码和 AI 项目有什么关系。后来我理解了：AI 应用不是孤立运行的，前端、后端、数据库和大模型之间都需要通过 API 传递数据。

4. 如果给客户讲，我会怎么说？

如果给客户讲，我会说：企业知识库 AI 助手需要让用户在网页上提问，然后把问题发送给后端，后端再调用知识库和 AI 服务生成回答，最后把结果返回给用户。API 就是这些系统之间沟通的桥梁。

5. 明天要解决什么？

明天我要学习 FastAPI，用 Python 写自己的后端接口。目标是做出一个 `/chat` 接口，用户提交问题后，后端返回一个固定回答。
